## Phase 1：MVP（M1-M3）

### 目标

验证核心假设：**"AI 生成的个股分析报告，比竞品的碎片化问答更有价值"。**

### 范围

| 包含 | 不包含 |
|------|--------|
| 股票搜索（代码/名称/拼音） | 实时行情/K线图 |
| 财务数据展示（图表+指标卡） | 行业对标分析 |
| AI 快速分析（一句话+亮点+风险） | 批量赛道扫描 |
| AI 报告生成（标准版，7章） | 自选股异动监控 |
| 自选股列表（添加/查看/删除） | 多市场（港美股） |
| 公告列表+AI摘要 | 用户系统（注册/登录/MVP单人用） |
| 报告导出（Word/PDF） | 移动端适配 |
| Web 端（PC优先） | 国际化 |

### 里程碑

- [ ] **M1.W2**：技术选型确认 + 开发环境搭建
  - 后端：FastAPI + SQLAlchemy 项目初始化，数据库表创建
  - 前端：React + Vite + Tailwind 项目初始化，路由配置
  - 数据：AkShare 对接完成，能拉到真实 A 股数据

- [ ] **M1.W4**：数据管线完成
  - 股票基础信息+财务数据定时拉取入库
  - 搜索接口（模糊匹配）完成
  - 股票详情接口（信息聚合）完成
  - 前端搜索页+详情页（静态展示，无AI）完成

- [ ] **M2.W2**：AI 分析完成
  - AI 快速分析接口对接（DeepSeek-V3）
  - System Prompt 调试完成（经过3轮以上迭代）
  - 详情页集成 AI 分析展示
  - 前端图表渲染（Recharts）

- [ ] **M2.W4**：报告生成完成
  - 报告生成接口（异步任务）
  - 报告查看页（左右分栏+Markdown渲染）
  - 报告导出（Word/PDF）
  - 自选股列表页

- [ ] **M3.W2**：内部测试
  - PM自己用 FinSight 研究5只股票，记录所有不爽的地方
  - 找3-5个朋友试用，收集反馈
  - Bug 修复 + 体验优化

- [ ] **M3.W4**：MVP 上线
  - Docker 部署到云服务器
  - 域名+HTTPS 配置
  - 可对外分享的 URL

## Phase 2：V1.0（M4-M5）

### 目标

增加差异化功能深度，从"能用"到"好用"。

### 新增功能

| 功能 | 原因 |
|------|------|
| 行业对标分析 | 用户画像林薇/赵明远的 top 需求：知道自己关注的股票在行业里处于什么位置 |
| 自选股异动监控 | 从"用户主动查"变为"系统主动推"，是留存率提升最直接的抓手 |
| 公告详情页 | MVP 只有公告列表+摘要，V1支持查看完整公告内容 |
| 用户系统（注册/登录） | 多人使用的前提，也是自选股数据跨设备的必要条件 |
| 报告重新生成 | 用最新数据重新生成同一只股票的报告 |
| 移动端基础适配 | 不是原生App，是响应式布局——让手机浏览器上能看 |

### 里程碑

- [ ] **M4.W2**：行业对标数据源接入 + 前端展示
- [ ] **M4.W4**：异动监控后端（定时任务+规则引擎）+ 前端通知
- [ ] **M5.W2**：用户系统（注册/登录/JWT鉴权）
- [ ] **M5.W4**：移动端响应式适配 + V1.0 上线

## Phase 3：V2.0（M6+）

### 目标

从"个股研究工具"扩展为"投资研究平台"。

### 新增功能

| 功能 | 原因 |
|------|------|
| 批量赛道扫描 | 赵明远场景的核心需求：输入行业概念 → 一键生成多股票快照对比 |
| 多市场支持（港股通） | 扩展用户覆盖面，很多个人投资者同时交易A股和港股 |
| 报告对比功能 | 两只股票并排对比分析（林薇场景） |
| 自定义分析模板 | 用户可调整报告包含/排除的章节（林薇进阶需求） |
| 数据导出 API | 给赵明远这类专业用户提供数据接口（商业化伏笔） |

| 角色 | 人数 | 说明 |
|------|:---:|------|
| 后端 + AI | 1（你自己） | Python + FastAPI + LLM接口 |

## 关键风险

| 风险 | 概率 | 影响 | 应对 |
|------|:---:|:---:|------|
| AkShare 接口不稳定/停更 | 中 | 高 | 准备 Tushare 作为备用数据源（需提前注册拿 token）；核心接口做好降级和缓存 |
| AI 分析质量不达预期 | 中 | 高 | 用规则引擎保底（财务数据展示至少是准确的）；AI 分析标注"仅供参考"；通过 prompt 迭代持续优化 |
| 一个人开发进度延迟 | 高 | 中 | 严格 MVP 范围，不做任何"顺手加的功能"；后端优先于前端完工（API好了前端可视情况简化） |
| DeepSeek API 涨价或不可用 | 低 | 高 | 保留 GPT-4o 和 Claude 接入能力（配置文件中可切换）；长期关注本地部署方案 |
| 竞品（同花顺）上线类似功能 | 中 | 中 | 差异化不在"有没有AI"而在"分析深度"——这是产品认知差，不是技术差。保持深度分析体验领先 |
| 数据合规风险（公告内容版权） | 低 | 中 | 仅展示摘要不存储全文；标注数据来源；法律咨询（正式商业化前） |
