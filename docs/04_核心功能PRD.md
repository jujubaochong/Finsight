# 04 核心功能 PRD

> 本文定义 MVP 的功能范围、核心流程、关键设计决策与边界情况。每个设计决策均附带取舍理由。

---

## 产品概述

- **产品名称**：FinSight
- **MVP 目标**：让用户输入 A 股代码后，获得"AI 快速分析 + 财务图表 + 结构化研报"的一站式投研体验
- **MVP 不做什么**：实时行情、交易下单、行业对标分析、批量赛道扫描、移动端 App

---

## 功能模块 1：股票搜索与信息聚合

### 功能描述

用户输入股票代码/名称/拼音 → 系统快速返回匹配结果 → 选择后进入股票详情页，看到多维信息汇总。

### 核心流程

```
用户输入搜索词
    │
    ├── 输入中（≥1字符）：debounce 300ms → 调用搜索接口 → 下拉展示匹配结果
    │       每条结果：代码 + 名称 + 市场 + 行业标签
    │
    ├── 回车 / 点击结果：直接跳转到股票详情页 /stock/{code}
    │
    └── 无匹配结果：展示"未找到相关股票，请检查代码或名称"
```

### 功能详述

**搜索输入支持**：

| 输入类型 | 示例 | 处理方式 |
|---------|------|---------|
| 6位代码 | 000001 | 精确匹配，直接返回 |
| 股票名称 | 平安银行 | 模糊匹配（包含即命中） |
| 拼音全拼 | pinganyinhang | 转汉字后模糊匹配 |
| 拼音首字母 | payh | 匹配拼音首字母索引 |
| 曾用名/简称 | 平安、PA | 匹配别名表 |

**信息聚合维度（股票详情页展示）**：

| 区域 | 内容 | 数据来源 |
|------|------|---------|
| 基本信息 | 代码、名称、市场、申万行业、上市日期、总市值 | 基础数据 |
| 核心财务指标卡 | 营收、净利润、毛利率、ROE、资产负债率（最近一期+同比变化） | 财务数据 |
| 近期公告摘要 | 近30天公告列表，每条含 AI 一句话摘要 | 公告数据 |
| AI 一句话总结 | 基于以上数据的综合概述 | AI 生成 |

### 设计决策

**Q: 为什么搜索后直接跳转详情页，而不是先展示搜索结果列表？**

A: 用户搜"平安银行"时，99% 就是想要平安银行的详情页，中间加一个列表页是多余步骤。唯一需要列表的场景是"搜了模糊词可能出现多个结果"（如搜"平安"可能匹配平安银行/中国平安），此时用下拉建议即可覆盖，不需要单独列表页。

**Q: 详情页为什么不展示实时行情/分时图/K线？**

A: 三个原因：
1. 行情数据需交易所授权，MVP 阶段不增加合规成本
2. 展示行情会将用户注意力引向"涨跌"而非"分析"，与我们"做基本面研究"的定位冲突
3. 用户手机上有同花顺看行情，我们做不过，不如不做

### 边界情况

| 情况 | 处理方案 |
|------|---------|
| 输入无效代码（如 999999） | 提示"未找到该股票代码，请检查" |
| 输入退市股票 | 展示基本信息 + 标注"该股票已退市"，不展示 AI 分析 |
| 搜索接口超时（>5秒） | 提示"网络繁忙，请重试"，保留已输入内容 |
| 输入为空 | 不触发搜索，展示引导文案 |
| 输入特殊字符 | 过滤特殊字符后搜索，仅保留字母/数字/中文 |
| 财务数据缺失（如新股无历史数据） | 对应区域标注"暂无足够历史数据"，其他区域正常展示 |

## 功能模块 2：AI 分析引擎

### 功能描述

AI 基于聚合的结构化数据，生成对股票的"快速分析"（详情页内嵌）+ "深度分析"（完整报告的基础）。

### 核心流程

```
触发：用户进入股票详情页
    │
    ▼
┌─────────────────────┐
│ 获取数据上下文       │
│ · 基本信息          │
│ · 近8季度财务数据   │
│ · 近30天公告        │
│ · 行业分类          │
└──────┬──────────────┘
       │
       ▼
┌─────────────────────┐
│ 快速分析（轻量）     │ ← 详情页立即展示，< 10秒
│ · 一句话总结        │
│ · 3个亮点          │
│ · 3个风险点        │
│ · 核心指标解读      │
└──────┬──────────────┘
       │
       ▼ (用户点击"生成完整报告")
┌─────────────────────┐
│ 深度分析（完整）     │ ← 异步任务，30秒-2分钟
│ · 完整财务分析      │
│ · 估值分析          │
│ · 风险清单          │
│ · 公告/新闻综合     │
└─────────────────────┘
```

### AI 分析维度

| 分析维度 | 具体内容 | 为什么用户需要 |
|---------|---------|-------------|
| **财务健康度** | 收入趋势、利润质量（经营现金流/净利润）、毛利率变化、ROE趋势 | 判断公司是否"真赚钱"而非纸面利润 |
| **成长性** | 收入增速、利润增速、是否可持续（增速来源拆解） | 判断增长是来自主业还是非经常性损益 |
| **风险信号** | 应收账款/营收比异常、存货周转恶化、大股东质押、关联交易、监管问询 | 主动发现用户可能忽略的危险信号 |
| **盈利质量** | 经营现金流与净利润偏离、应收账款增速 vs 收入增速 | 识别"纸面富裕"——利润表好看但现金流差 |
| **近期异动** | 近30天公告中值得关注的事件：减持、诉讼、资产重组、业绩预告修正 | 确保用户不会因为没看公告而踩雷 |

### 设计决策

**Q: 为什么分"快速分析"和"深度分析"两层？**

A: 对应场景1（初次了解）和场景2（深度复盘）。用户首次看一只股票时只需要快速判断"值不值得继续看"，10秒内出结果比分析深度更重要。只有当用户确认有兴趣时，才启动分钟的深度分析。另外，快速分析的计算成本远低于深度分析（约 1/5 token 消耗）。

**Q: AI 分析的语气怎么把握？**

A: 三条原则：
1. **客观呈现，不做投资建议**。"该公司毛利率从 35% 下降至 28%，低于行业均值 32%"——这是客观陈述。"该公司毛利率持续恶化，建议卖出"——这是违规。
2. **宁可说"需核实"也不强行下结论**。当证据不够充分时，标注"该信号需结合其他信息综合判断"。
3. **风险用温和措辞**。"值得关注的是……"而非"危险！"——避免引发用户恐慌性操作。

**Q: 怎么防止 AI 幻觉？**

A: 三层防线：
1. **数字只引用不出生**：System Prompt 强制要求所有财务数字必须从提供的结构化数据中引用原文，不允许 AI 自行计算或推测数字。
2. **来源标注**：每个数据引用标注来源（"2025Q1财报——利润表"），用户可追溯到原始数据。
3. **置信度标记**：AI 判断型内容（如"毛利率下降的原因可能是原材料涨价"）标注为"AI推断"，与数据事实明确区分。

## 功能模块 3：报告生成

### 功能描述

用户点击"生成完整报告" → 系统异步生成结构化研报 → 用户查看/编辑/导出。

### 报告结构（7章）

```
第一章：公司概览
  1.1 基本信息（代码、全称、行业、上市日期、总市值）
  1.2 主营业务描述（AI提炼）
  1.3 股权结构（实际控制人、控股股东）

第二章：财务分析
  2.1 收入与利润趋势（近8季度图表 + AI解读）
  2.2 盈利能力（毛利率、净利率、ROE趋势 + 解读）
  2.3 成长性（同比增速 + 增长质量判断）
  2.4 现金流分析（经营/投资/筹资现金流结构）
  2.5 资产负债结构（关键科目变化）

第三章：AI 风险识别
  3.1 财务风险（按等级排列：高/中/低）
  3.2 经营风险（客户集中度、行业政策等）
  3.3 合规与法律风险（监管问询、诉讼、股权质押）
  3.4 风险矩阵汇总表

第四章：近期重要事件
  4.1 近3个月公告摘要（AI提炼每条公告的核心信息）
  4.2 重要舆情/新闻（如有）

第五章：机构观点参考
  5.1 近6个月券商研报核心观点（如有数据源）
  5.2 市场一致预期（营收/利润预测）

第六章：估值参考
  6.1 当前估值水平（PE/PB/PS + 历史分位）
  6.2 估值说明（AI对当前估值的客观描述，不可做"高估/低估"判断）

第七章：总结
  7.1 公司核心优势（3-5点，客观陈述）
  7.2 值得关注的风险（按重要性排列）
  7.3 AI 分析说明（数据来源、分析方法、免责声明）
```

### 核心流程

```
用户点击 [生成完整报告]
    │
    ▼
后端创建异步任务 → 返回 task_id
    │
    ▼
前端轮询（每3秒）: GET /api/reports/{task_id}/status
    │
    ├── status: "processing" → 展示进度条 + 当前正在生成的章节
    │
    └── status: "completed" → 返回报告内容 → 渲染查看页
```

### 设计决策

**Q: 报告应该多长？**

A: 提供三种深度可选：
- **快览版**（约5页A4）：公司概览 + 财务走势图 + 关键风险清单。适合张弛。
- **标准版**（约15-20页）：完整7章。默认选项。
- **深度版**（约30页+）：增加分部业务分析、详细估值模型。适合林薇和赵明远。

MVP 先做标准版。快览版本质上是详情页的 AI 分析（已有）。深度版依赖行业对标和更多数据源（V1）。

**Q: 用户能编辑报告吗？**

A: MVP 阶段：报告为只读展示 + 用户可导出 Word 后自由编辑。V1 阶段考虑在浏览器内支持简单编辑（类似 Notion 体验），但 MVP 不做——编辑器的开发量不亚于报告生成本身。

## 功能模块 4：自选股管理

### 功能描述

用户将关注的股票加入自选列表，系统在有重要变化时主动提醒。

### 核心流程

```
添加自选：
  方式1：股票详情页 → 点击 [加入自选]
  方式2：自选股页面 → 搜索股票 → 添加

查看自选：
  自选股页面 → 列表展示 → 点击进入详情

异动通知：
  定时任务扫描自选股 → 匹配异动规则 → 系统通知
```

### 异动规则定义

| 异动类型 | 触发条件 | 优先级 |
|---------|---------|:----:|
| 重大公告 | 发布年报/半年报/季报 | P0 |
| 重大公告 | 发布业绩预告（预增/预减>50%） | P0 |
| 股东行为 | 大股东减持 >1% | P1 |
| 股东行为 | 实际控制人变更 | P0 |
| 监管风险 | 收到交易所问询函/关注函 | P0 |
| 经营风险 | 重大诉讼、资产冻结 | P1 |
| 公司治理 | 董监高大比例辞职 | P2 |
| 股权质押 | 大股东质押比例 >80% | P1 |

### 设计决策

**Q: 为什么不监控"涨跌幅 >5%"？**

A: 那是行情异动，需要实时行情数据（MVP不做）。而且股票涨跌不是 FinSight 的核心关注点——我们关心的是"基本面发生了什么变化"，不是"价格发生了什么变化"。一个做基本面研究的工具整天推送价格异动，会让用户困惑"你到底是个什么产品"。

**Q: 推送频率怎么控制？**

A: 每条规则每天最多推送1次。同一股票同一天的多条异动合并为一条通知："平安银行今日发布年报 + 收到问询函"。

## 非功能性需求

| 需求项 | MVP 目标 | 说明 |
|--------|:------:|------|
| 搜索响应时间 | < 500ms | 含模糊匹配 + DB查询 |
| AI 快速分析时间 | < 10秒 | 详情页加载时并行请求 |
| 报告生成时间 | < 2分钟 | 标准版报告，异步任务 |
| 页面首屏加载 | < 2秒 | 含图表渲染 |
| 财务数据更新频率 | 每个交易日更新 | 收盘后拉取最新财务数据 |
| 公告数据更新频率 | 每30分钟 | 盘中也可能有公告 |
| MVP 并发支持 | 50 用户同时在线 | 单机部署即可满足 |
| 数据安全 | HTTPS + 数据库加密 + API Key 环境变量隔离 | 不存储用户投资组合金额等敏感数据 |
| 可用性 | 99% uptime | 非金融交易系统，允许偶尔维护 |
