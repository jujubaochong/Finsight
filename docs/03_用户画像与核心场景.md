# 03 用户画像与核心场景

> 用户画像基于真实用户的观察与访谈提炼，强调可被想象的具体生活场景，而非抽象的功能清单。

---

## 用户画像 1：张弛 — "想研究但不会研究"

```
基本信息：
  姓名：张弛
  年龄：31岁
  职业：互联网公司产品经理
  投资经验：买过基金，2024年开始自己买股票
  投资风格：想走价值投资路线，但实际偏"有逻辑的跟风"
  A股投资年限：2年
  管理资金：约 40 万（个人积蓄+工资定投）

典型的一天：
  早上通勤地铁上，张弛打开同花顺刷了刷自选股，几只股票涨跌互现，他看了看
  K线图但看不出什么名堂。到了公司，同事在群里发"宁德时代最近不错啊，
  固态电池要量产了"，张弛心想上周也听人提过，决定研究一下。

  打开同花顺搜"宁德时代"→ F10 资料页 → 几十个财务指标密密麻麻排列，
  他不知道该看哪个。切到"财务分析"tab，看到近5年收入利润走势图，曲线
  确实在涨。但他心里没底：这个增速算好还是不好？跟同行比呢？有没有我没
  看到的坑？

  又打开雪球搜宁德时代，想看看大家怎么说。评论区前20条：5条在吵架（
  "宁王还是宁亡？"），8条在吹票，3条在骂，4条在讲K线技术指标。翻了
  十分钟也没找到一份靠谱的基本面分析。

  最后他打开微信，给一个做投资的朋友发了条消息："宁德现在能入吗？"
  朋友回复："我看过但没深研，你最好自己看看财报。"

  张弛关掉手机，心想：算了，还是买点沪深300 ETF 吧。

核心痛点（按疼痛程度排）：
  1. 不知道该看什么 —— 面对100+个财务指标，没有"分析路径"引导
  2. 看了也不知道意味着什么 —— 毛利率35%是好事还是坏事？缺少同行对标
  3. 找不到靠谱的二手分析 —— 雪球噪音太大，研报太专业看不太懂
  4. 每次研究都像第一次 —— 没有模板、没有checklist、没有积累

一句话描述他：
  "他不是不想研究，是被工具门槛劝退了。他要的不是更多数据，
   是一个'帮他读过财报的人'告诉他重点在哪。"
```

### 张弛怎么用 FinSight

| 行为 | 详情 |
|------|------|
| **触发** | 听到"宁德时代" → 打开 FinSight 输入代码 |
| **操作** | 先看 AI 一句话摘要 → 展开看亮点和风险 → 对风险部分特别关注 |
| **决策** | 如果 AI 的风险提示里有他不了解的事项（比如"应收账款异常增长"），他可能会放弃投资或进一步核实 |
| **留存行为** | 把觉得不错的股票加入自选，设定异动提醒 |

## 用户画像 2：林薇 — "半专业独立研究者"

```
基本信息：
  姓名：林薇
  年龄：37岁
  职业：财务咨询公司合伙人（注册会计师背景）
  投资经验：10年A股投资，经历过一轮完整牛熊
  投资风格：深度价值投资，偏好消费和医药行业，持股周期1-3年
  A股投资年限：10年
  管理资金：约 200 万（家庭资产，非代客理财）

典型的一天：
  林薇每个季度都会做一个固定功课：重读所有持仓股的季报，更新自己的
  估值模型。这个过程通常需要1-2个完整的周末。

  周六早上，林薇泡了杯咖啡打开电脑。她先打开东方财富，把贵州茅台的
  最新季报PDF下载下来。然后打开自己维护的Excel模型，开始逐页看财报——
  营收多少、成本结构有没有变化、现金流是否健康、各项比率是否在合理
  区间。每读到一个数据就手动填入Excel，计算同比环比。

  看了30分钟后她发现茅台的预收账款（合同负债）环比大幅下降，心里
  一紧——这是一个她需要重点关注的风险信号。但如果不是她恰好知道要
  看这个科目，很可能就略过去了。

  下午她打开雪球，想看看有没有关于茅台的最新深度分析。翻了很久，
  找到一篇不错的帖子，作者提到了经销商库存高企的问题，这和她观察到
  的预收款下降形成了交叉验证。

  晚上她开始写投资笔记，把手写的Excel数据和阅读心得整理成一段文字。
  整个过程耗时约6小时，覆盖了3只持仓股的季报。

核心痛点（按疼痛程度排）：
  1. 手动搬数据效率极低 —— 从PDF到Excel到笔记，大量机械劳动
  2. 怕漏掉关键信号 —— 财报200页，只有一个分析师（她自己），
     总有看漏的风险
  3. 行业对标数据难获取 —— 想知道"茅台毛利率波动是否正常"，
     需要手动查同行业公司，很耗时
  4. 研究无法批量复用 —— 每次季报季都要从头来一遍，没有模板化

一句话描述他：
  "她有分析能力，但工具还停留在Excel+PDF时代。
   她需要的不是别人替她判断，是把'搬数据、算比率、找对标'
   这些机械工作自动化，让她聚焦在真正的分析上。"
```

### 林薇怎么用 FinSight

| 行为 | 详情 |
|------|------|
| **触发** | 季报发布 → 打开 FinSight 搜索持仓股 |
| **操作** | 看 AI 自动提取的财务数据+图表 → 重点关注风险识别部分 → 发现有价值的风险点就展开看证据链 → 生成完整研报作为投资笔记 |
| **决策** | 如果 AI 未发现她关心的风险信号，手动补充到报告中；如果 AI 发现了她没想到的风险信号，标记为重点核查项 |
| **留存行为** | 每季度固定使用，持仓股的研报全部由 FinSight 生成并归档 |

## 用户画像 3：赵明远 — "没有专业工具的专业用户"

```
基本信息：
  姓名：赵明远
  年龄：26岁
  职业：小型FA（财务顾问）机构分析师
  投资经验：2年正式分析工作，覆盖消费和TMT赛道
  投资风格：服务PE/VC客户，产出标的研究和行业扫描
  A股投资年限：从业2年
  管理资金：不直接管理，但经手的项目单笔 5000万-3亿

典型的一天：
  赵明远的老板周一给了他一个任务："看看A股预制菜赛道，找3-5家值得
  关注的标的，这周五给我一版初筛报告。"

  他们公司没有买Wind（太贵了，小FA负担不起），所以他打开东方财富
  Choice的基础版。先搜行业分类，找到预制菜概念股，大约有15家。
  接下来他要逐一过一遍：每家公司的收入规模、增速、毛利率、净利率、
  大客户依赖度、近期有没有重大公告。

  看第一家公司就花了20分钟：打开F10→财务分析→手动摘数据到Excel→
  打开巨潮资讯搜公司公告→打开雪球看讨论。15家公司看一遍，按这个
  速度需要5小时以上。

  最痛苦的是对比环节：他需要手动把15家公司的核心指标放在一张表里，
  按增速排序、标出异常值。这个过程纯粹是"人肉ETL"。

  周三晚上他加班到11点才把初筛报告写完。老板周四反馈："这几家你
  有没有注意到他们Q3的应收账款都异常增长？"赵明远心里一沉——这个
  他没注意到，因为15家公司挨个看已经耗光了他的时间。

核心痛点（按疼痛程度排）：
  1. 批量分析效率极低 —— 没有工具支持"一次看一个赛道"
  2. 缺少系统化分析框架 —— 每次自己定分析维度，容易遗漏
  3. 报告撰写耗时 —— 数据收集后还要手工组织成文
  4. 无法负担专业工具 —— Wind年费3-8万，是薪资之外的公司成本

一句话描述他：
  "他在做专业投资分析的工作，但手里的工具是散户级别的。
   他能分辨好坏分析，但没时间做每一只股票的'数据搬家'工作。"
```

### 赵明远怎么用 FinSight

| 行为 | 详情 |
|------|------|
| **触发** | 老板布置赛道扫描任务 → 打开 FinSight 逐一搜索 |
| **操作** | 对每家标的生成一份标准研报 → 横向对比 → 标注关键差异 → 导出为初筛底稿 |
| **决策** | AI 风险提示帮他快速排除有硬伤的公司，聚焦在2-3家值得深研的标的 |
| **留存行为** | 每个项目都会用，可能是重度用户 |

## 三个核心使用场景

### 场景 1：初次了解一只股票（"这是什么？"）

**触发条件**：用户从任何渠道（朋友推荐、新闻、群聊、自选观察）听说一只股票，想快速了解这家公司。

**当前做法**：
1. 打开同花顺/东财 → 看K线 → 看F10（3-5分钟，获得碎片化印象）
2. 打开雪球 → 搜代码看讨论（5-10分钟，信噪比低）
3. 如果感兴趣 → 下决心找财报看（大部分人在这步放弃）

**期望行为**：
1. 打开 FinSight → 输入代码 → 5秒内看到 AI 一句话总结
2. 30秒读完"亮点+风险" → 决定是否有兴趣继续深看
3. 如果有兴趣 → 展开看财务趋势图表和 AI 解读 → 5分钟形成基本判断
4. 如果很有兴趣 → 点击"生成完整报告" → 获取深度研报

**成功标准**：用户在 5 分钟内对这只股票形成了"有数据支撑的基本判断"（而非"大概知道是干嘛的"）。

---

### 场景 2：财报季定期复盘（"我的持仓还好吗？"）

**触发条件**：年报/季报密集披露期（每年3-4月和8-9月），用户需要集中审视持仓股的财务变化。

**当前做法**：
1. 下载每只持仓股的财报PDF（5-10分钟/只）
2. 逐只翻阅，手动对比上期数据（20-30分钟/只）
3. 发现异常后手动做标记，再查原因（不确定性大）
4. 通常只能覆盖2-3只股票，其余"等有空再看"

**期望行为**：
1. 打开 FinSight → 从自选股列表逐一进入详情页
2. 直接看"本期变化亮点/风险"——AI 自动对比上期数据，标注异常变动
3. 对标记的风险点展开看证据链 → 判断是否需要减仓或进一步核查
4. 对每只股票生成一份季度跟踪报告 → 归档为投资笔记

**成功标准**：用户能在 1-2 小时内完成全部持仓的季度回顾（当前需1-2天）。

---

### 场景 3：赛道扫描（"这个行业有哪些好标的？"）

**触发条件**：用户想研究一个行业/主题（如固态电池、低空经济、预制菜），需要找到值得关注的标的。

> 注：此场景 MVP 阶段不完全支持，属于 V1 规划，此处列出以保证场景体系的完整性。

**当前做法**：
1. 在东方财富按行业分类找到概念股列表
2. 逐一打开每家公司的F10，手动摘数据到Excel（人肉ETL）
3. 凭经验判断哪些公司值得深看
4. 耗时N×20分钟（N=概念股数量），通常N=10-20家公司

**期望行为**：
1. 输入行业/概念关键词 → 系统列出相关股票
2. 批量生成"一页纸快照"（每家公司核心指标+亮点+风险）
3. 横向对比 → 快速排除有明显问题的
4. 对剩余2-3家重点标的生成深度报告

**成功标准**：用户能在 1-2 小时内完成一个赛道的初筛（当前需1-2天）。

## 非目标用户

| 不是我们的用户 | 为什么 | 他们该用什么 |
|--------------|--------|------------|
| **纯短线交易者** | 只看K线、量价、资金流向，不关心基本面。FinSight对他们来说是"信息过载"而非"信息增值" | 同花顺 Level-2、通达信 |
| **打板/追涨停族** | 决策窗口是分钟级，不需要研报 | 同花顺、东方财富 |
| **机构基金经理** | 有 Bloomberg/Wind 终端 + 卖方研报服务，工具链完整。不会用一个 ToC 产品做投资决策 | Wind、Bloomberg |
| **只买基金/ETF的人** | 不研究个股 | 蚂蚁财富、天天基金 |
| **纯KOL跟单者** | 投资决策 = "大V买了没"，不需要自己分析 | 雪球、微博、微信群 |

> 明确非目标用户的意义：当出现"能否增加 K 线图"这类需求时，可据此判断——该需求来自非目标用户，不应为此改变产品方向。

## 需求优先级（MoSCoW）

| 优先级 | 需求 | 理由 |
|:------:|------|------|
| **Must** | 输入代码→AI快速分析（一句话总结+亮点+风险） | 场景1的核心体验，没有这个产品不成立 |
| **Must** | 财务数据可视化（收入/利润/利润率趋势图） | 场景1和2都依赖，让用户"看到"数据而不仅是"读到" |
| **Must** | AI生成完整研报（结构化报告，非碎片回答） | 和竞品的核心差异点，三个场景都依赖 |
| **Must** | 自选股管理（添加/移除/列表查看） | 场景2的前提，也是留存的抓手 |
| **Should** | 异动提醒（公告/新闻自动推送） | 场景2的增强，让用户从"主动查"变"被动收" |
| **Should** | 报告导出（PDF/Word） | 场景2和3的收尾环节，尤其赵明远这种专业用户需要 |
| **Should** | 财务指标行业对标（你的毛利率 vs 行业均值） | 林薇场景的核心需求，但依赖行业数据源，MVP可延后 |
| **Could** | 批量赛道扫描（输入行业→多股票快照对比） | 场景3，开发量大，MVP不做 |
| **Could** | 用户自定义分析模板 | 林薇的高级需求，V2考虑 |
| **Won't** | 实时行情/K线图 | 不是我们的核心价值，做好也做不过同花顺 |
| **Won't** | 交易下单 | 合规门槛，也没有差异化 |
| **Won't** | 社区/UGC | 冷启动不可能，且雪球壁垒已形成 |
